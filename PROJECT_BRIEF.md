## 1. КОНТЕКСТ И ПРОБЛЕМА

### 1.1 Ситуация
Есть работающий сервис личных кабинетов на сайте Тильда, построенный на связке иностранных no-code инструментов. В связи с уходом иностранных сервисов из РФ необходимо мигрировать на self-hosted решение, которое будет развёрнуто на собственном сервере.

### 1.2 Текущий стек

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              ТЕКУЩАЯ АРХИТЕКТУРА                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────┐                      ┌──────────────┐      ┌─────────────┐   │
│   │  Tilda  │ ──── Webhooks ─────► │     Make     │ ───► │   Airtable  │   │
│   │  (сайт) │                      │ (Integromat) │      │   (база)    │   │
│   └────┬────┘                      └──────────────┘      └──────┬──────┘   │
│        │                                                         │          │
│        │              ┌──────────────┐                           │          │
│        │              │   Collabza   │ ◄─────── API ─────────────┘          │
│        └─────────────►│   (виджет)   │                                       │
│                       └──────────────┘                                       │
│                              │                                               │
│                              ▼                                               │
│                       Рендеринг данных в личном кабинете                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 Описание текущих сервисов

**Tilda (остаётся)**
- Конструктор сайтов, на котором построен основной сайт
- Формы для сбора данных пользователей
- Страницы личных кабинетов (HTML-блоки для встраивания виджетов)
- Webhooks для отправки данных при сабмите форм

**Collabza (нужно заменить)**

Что делает:
- JavaScript-виджет, встраиваемый в HTML-блок Тильды
- Подключается к Airtable через API
- Авторизует пользователей (email/пароль)
- Рендерит данные на странице в различных форматах (таблицы, карточки, списки, формы с фильтрами)
- Позволяет создавать личные кабинеты без кода
- Документация: https://collabza.ru/docs

**Make / Integromat (нужно заменить)**

Что делает:
- Получает webhooks от Тильды при отправке форм
- Обрабатывает и трансформирует данные
- Записывает данные в Airtable
- Отправляет уведомления (email, Telegram)
- Синхронизирует данные между таблицами

**Airtable (нужно заменить)**

Что делает:
- Облачная база данных с UI как у Excel
- Хранит все данные системы
- 3 таблицы со связями между ними
- API для доступа к данным

## 3. ПОДРОБНОЕ ОПИСАНИЕ NOCODB

### 3.1 Что такое NocoDB

NocoDB — open-source альтернатива Airtable с 50,000+ звёзд на GitHub. Превращает любую SQL базу данных (PostgreSQL, MySQL, SQLite) в удобный интерфейс электронных таблиц с полноценным REST API.

**Ключевые преимущества:**
- ✅ API максимально похож на Airtable
- ✅ Импорт данных из Airtable есть из коробки
- ✅ Webhooks для автоматизаций
- ✅ Ролевая модель доступа
- ✅ Самохостинг — полный контроль над данными

- **GitHub:** https://github.com/nocodb/nocodb
- **Документация:** https://docs.nocodb.com

### 3.2 Сравнение API Airtable vs NocoDB

**Airtable API:**
```javascript
// Получить записи
GET https://api.airtable.com/v0/{baseId}/{tableName}
Headers: Authorization: Bearer {apiKey}

// Создать запись
POST https://api.airtable.com/v0/{baseId}/{tableName}
Body: { "fields": { "Name": "Test" } }
```

**NocoDB API:**
```javascript
// Получить записи
GET http://your-server:8080/api/v1/db/data/v1/{projectId}/{tableName}
Headers: xc-token: {apiToken}

// Создать запись
POST http://your-server:8080/api/v1/db/data/v1/{projectId}/{tableName}
Body: { "Name": "Test" }
```

### 3.3 Docker Compose для запуска NocoDB

```yaml
version: '3.8'

services:
  nocodb:
    image: nocodb/nocodb:latest
    container_name: nocodb
    ports:
      - "8080:8080"
    environment:
      NC_DB: "postgresql://nocouser:nocopass@postgres:5432/nocodb"
      NC_PUBLIC_URL: "http://your-domain.ru"
      NC_AUTH_JWT_SECRET: "your-jwt-secret-change-this"
    depends_on:
      - postgres
    volumes:
      - nocodb_data:/usr/app/data
    restart: unless-stopped

  postgres:
    image: postgres:15-alpine
    container_name: nocodb_postgres
    environment:
      POSTGRES_USER: nocouser
      POSTGRES_PASSWORD: nocopass
      POSTGRES_DB: nocodb
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

volumes:
  nocodb_data:
  postgres_data:
```

## 4. ТРЕБОВАНИЯ К РАЗРАБОТКЕ

### 4.1 JS-виджет для Тильды (аналог Collabza)

**Функциональные требования:**

1. **Авторизация пользователей**
   - Форма логина (email + пароль)
   - Восстановление пароля
   - Сохранение сессии (localStorage / cookies)
   - Автоматический логаут по истечении сессии

2. **Получение данных из NocoDB**
   - Загрузка профиля пользователя
   - Загрузка связанных данных (заказы, курсы)
   - Фильтрация и сортировка
   - Пагинация для больших списков

3. **Рендеринг данных**
   - Таблицы с сортировкой
   - Карточки с детальной информацией
   - Формы редактирования профиля
   - Pop-up модальные окна

4. **Интеграция с Тильдой**
   - Встраивание через HTML-блок
   - Адаптация под дизайн сайта
   - Мобильная адаптация

**Технические требования:**
- Vanilla JavaScript или Vue.js 3 (CDN)
- Минимальные зависимости
- Работает в iframe Тильды
- CORS-friendly

**Структура виджета:**
```
widget/
├── index.html          # Точка входа
├── css/
│   └── styles.css      # Стили
├── js/
│   ├── app.js          # Главный модуль
│   ├── api.js          # Работа с NocoDB API
│   ├── auth.js         # Авторизация
│   ├── render.js       # Рендеринг компонентов
│   └── utils.js        # Утилиты
└── config.js           # Конфигурация
```

### 4.2 Webhook Handler (если NocoDB недостаточно)

**Функциональные требования:**
- Приём webhooks от Тильды (парсинг payload, валидация)
- Запись в NocoDB (создание/обновление записей)
- Дополнительно: Telegram уведомления, логирование

| Дата продления | DateTime | Дата продления тарифа | 8.10.2023 17:39 |
| Кто привёл | Link to Users | Реферал (связь с Users) | recfwm7dTrwhc5yWO |
| Реферальная ссылка1 | Formula/Long Text | Сгенерированная ссылка | https://e-budget.ru/members/signup?utm_source=... |
| Покупки | Link to Покупки | Связь с таблицей покупок | recWu0BDtWeRnT4Jx |
| Курсы_link | Link to Курсы | Связь с таблицей курсов | "Заключение соглашений..." |

| Оплатили подписку | Number | Сколько рефералов оплатили | 0 |

## 10. ТЕХНИЧЕСКИЕ ДЕТАЛИ

### 10.1 Особенности данных

1. **Реферальная система**
   - Поле "Кто привёл" в Users — это связь на другого пользователя
   - При регистрации по реферальной ссылке создаётся связь
   - Бонусы начисляются при покупке реферала

2. **Бонусная система**
   - Бонусы начисляются в поле "Бонусы начислить" в таблице Покупки
   - Накопленные бонусы хранятся в поле "Бонусы" в Users
   - Бонусы можно списать при следующей покупке

3. **HTML/JS код в базе**
   - Поля `html_psy`, `Code_block` содержат готовый JS код
   - Этот код Collabza использует для рендеринга
   - В новом виджете нужно заменить на шаблоны

### 10.2 Миграция данных

**Что нужно мигрировать:**
- ~15-20 пользователей
- ~6 курсов
- ~19 покупок

**Как мигрировать:**
1. Экспортировать CSV из Airtable (уже есть!)
2. Импортировать в NocoDB через UI
3. Настроить связи между таблицами
4. Проверить корректность данных
