# Настройка формул в NocoDB

## Синтаксис формул NocoDB

NocoDB использует синтаксис, похожий на Excel/Google Sheets:
- Конкатенация: `CONCAT(text1, text2, ...)` или `&`
- Условия: `IF(condition, true_value, false_value)`
- Ссылки на поля: `{Имя поля}`
- Строки в кавычках: `"текст"`

**ВАЖНО:** NocoDB не поддерживает многострочные формулы и сложные HTML с кавычками внутри!

---

## Таблица Users — Формульные поля

### 1. html_psy (HTML код для вставки имени)

**Назначение:** Вставляет имя пользователя в элемент `.fio .tn-atom` на странице Tilda

**Формула NocoDB:**
```
CONCAT("<script>$(\".fio .tn-atom\").html(\"", IF({Имя}, {Имя}, ""), "\");</script>")
```

**Как настроить:**
1. Открой таблицу Users в NocoDB
2. Найди поле `html_psy` (или создай новое)
3. Edit Column → Column Type: `Formula`
4. Вставь формулу выше
5. Save

---

### 2. Реферальная ссылка1 (Реферальная ссылка с utm_source)

**Назначение:** Генерирует уникальную реферальную ссылку для пользователя

**Формула NocoDB:**
```
CONCAT("https://e-budget.ru/members/signup?utm_source=", {ID}, "&utm_medium=referral&utm_campaign=friends")
```

**Альтернатива с HTML кодом (как в Airtable):**
```
CONCAT("<script>$('.example-1 .tn-atom').text('https://e-budget.ru/members/signup?utm_source=", {ID}, "&utm_medium=referral&utm_campaign=friends>');</script>")
```

---

### 3. Tilda Статус подписки (HTML код статуса тарифа)

**Назначение:** Отображает текущий тариф пользователя

**Формула NocoDB:**
```
CONCAT("</br></span><b>Текущий тариф:</b> ", IF({Тариф}, {Тариф}, "🥈Базовый"), "<br><a href='https://e-budget.ru/pro'>Улучшить тариф ➤</a></br>")
```

---

### 4. status_prof (HTML код для вставки тарифа)

**Назначение:** JavaScript код для вставки тарифа на страницу

**Проблема:** NocoDB не поддерживает сложные многострочные формулы с экранированием кавычек!

**Решение:** Упрощённая версия или использовать Lookup/Rollup

**Упрощённая формула:**
```
CONCAT("$(\".tarif .tn-atom\").html(\"", {Tilda Статус подписки}, "\");")
```

### 5. Code_block (HTML код для localStorage)

**Назначение:** Устанавливает статус PRO в localStorage и скрывает иконку sale

**Формула NocoDB:**
```
IF({Тариф} = "🥇Про", 
  "<script>localStorage.setItem(\"pro\", 1);$(\".saleicon\").show();</script>",
  "<script>localStorage.setItem(\"pro\", 0);$(\".saleicon\").hide();</script>"
)
```

### 6. Dashboard (HTML код дашборда)

**Назначение:** Отображает количество курсов, рефералов и бонусов

**Формула NocoDB:**
```
CONCAT(
  "<script>",
  "$(\".tn-elem__5435214961675016424996 .tn-atom\").text(", 
  IF({Количество рефералов}, {Количество рефералов}, 0), 
  ");",
  "$(\".tn-elem__5435214961675016455904 .tn-atom\").text(", 
  IF({Оплатили подписку}, {Оплатили подписку}, 0), 
  ");",
  "$(\".tn-elem__5435214961676984464176 .tn-atom\").text(", 
  IF({Бонусы}, {Бонусы}, 0), 
  "+\" б.\");",
  "</script>"
)
```

**ВНИМАНИЕ:** Эта формула может не работать из-за сложности. Если не работает, используй Вариант B (генерация в API).

### 7. HTML_ADD (Комбинированный HTML)

**Назначение:** Объединяет все HTML коды в один

**Формула NocoDB:**
```
CONCAT(
  {Dashboard},
  {Реферальная ссылка1},
  {Списать бонусы},
  {Code_block}
)
```

---

### 8. Списать бонусы (HTML кнопка списания бонусов)

**Назначение:** Создаёт кнопку для списания бонусов в корзине Tilda

**Формула NocoDB (упрощённая):**
```
CONCAT(
  "<script>function applyBonus(){$(\".t-btn__bonus\").remove();promo={promocode:\"BONUS\",discountsum:Math.min(window.tcart.amount-1,",
  IF({Бонусы}, {Бонусы}, 0),
  ")};t_input_promocode__addPromocode(promo);}$('<div onclick=\"applyBonus();\" class=\"t-btn t-btn__bonus\" style=\"width:100%;\">Списать бонусы (доступно ",
  IF({Бонусы}, {Бонусы}, 0),
  ")</div>').insertBefore(\".t706 .t-form__submit\");</script>"
)
```

**ВНИМАНИЕ:** Эта формула очень сложная и может не работать в NocoDB из-за ограничений на экранирование кавычек!

## Проблемы с Вариантом A (формулы в NocoDB)

### Ограничения NocoDB:

1. **Экранирование кавычек:** Сложно работать с HTML/JS кодом, где много `"` и `'`
2. **Длина формулы:** Очень длинные формулы могут не работать
3. **Многострочность:** Нельзя делать читаемые многострочные формулы
4. **Отладка:** Сложно найти ошибку в длинной формуле

### Рекомендация:

Для **простых полей** (Реферальная ссылка, Tilda Статус подписки) — используй формулы.

Для **сложных HTML/JS кодов** (Dashboard, Списать бонусы, HTML_ADD) — лучше использовать **Вариант B** (генерация в API).

## Пошаговая настройка в NocoDB

### Шаг 1: Настрой простые формулы

1. Зайди в NocoDB → таблица `Users_Grid_view_csv`
2. Для каждого поля:
   - Кликни на заголовок столбца
   - Edit Column
   - Column Type: `Formula`
   - Вставь формулу
   - Save

### Шаг 2: Проверь результат

1. Открой любую запись пользователя
2. Проверь, что формулы работают
3. Если видишь ошибку — упрости формулу или используй API

### Шаг 3: Для сложных полей используй API

Если формулы не работают, я обновлю API для генерации этих полей.
